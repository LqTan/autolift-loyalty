# AutoLift ML Pipeline

Uplift Modeling pipeline for Autolift Loyalty System.

## Structure

```
ml/
├── autolift_ml/              # Python package
│   ├── pipeline/             # Pipeline steps
│   │   ├── build_features.py
│   │   ├── train.py
│   │   ├── evaluate.py
│   │   └── export.py
│   └── utils/                # Utilities
│       ├── data_loader.py
│       ├── metrics.py
│       └── feature_config.py
├── scripts/
│   └── run_pipeline.py       # CLI entry point
├── data/                      # Raw X5 dataset (read-only input)
├── artifacts/                 # Pipeline outputs
│   ├── models/                # Trained models (.joblib)
│   ├── outputs/               # CSVs for Spring Boot import
│   └── cache/                 # Intermediate features (x5_customer_features_*.csv)
├── notebooks/                 # Experiments
├── pyproject.toml             # Package metadata
├── requirements.txt
└── config.yaml               # Configuration
```

## Directory Purpose

| Directory | Purpose | Can Delete? |
|-----------|---------|-------------|
| `data/` | Raw X5 gz files (input) | No - re-download required |
| `artifacts/models/` | Trained .joblib models | No - retrain required |
| `artifacts/outputs/` | Final CSV for Spring Boot | No - Spring Boot imports from here |
| `artifacts/cache/` | Intermediate features | Yes - regenerated by pipeline |

## Setup

```bash
# 1. Create conda environment (one time only)
conda create -n autolift-ml python=3.11 -y
conda activate autolift-ml

# 2. Install dependencies (one time only)
pip install -r ml/requirements.txt

# 3. Install package in development mode (one time only)
cd ml && pip install -e .
```

## Running the Pipeline

```bash
# Activate environment first
conda activate autolift-ml

# Run full pipeline (will auto-download X5 dataset ~600MB)
python ml/scripts/run_pipeline.py

# With options
python ml/scripts/run_pipeline.py --no-download
python ml/scripts/run_pipeline.py --help
```

## Output Files

### For Spring Boot Import
- `ml/artifacts/outputs/customer_uplift_scores.csv` - Uplift scores for targeting
- `ml/artifacts/outputs/customer_feature_snapshots.csv` - Feature snapshots

### Models
- `ml/artifacts/models/model_treatment_v1.joblib` - Treatment model
- `ml/artifacts/models/model_control_v1.joblib` - Control model

## Pipeline Steps

1. **Download** - Fetch X5 RetailHero dataset with MD5 verification
2. **Build Features** - Aggregate purchase data (chunked 300K rows)
3. **Train** - Train T-Learner uplift model (treatment + control)
4. **Evaluate** - Calculate Qini/AUUC/Uplift@K metrics
5. **Export** - Save CSV files for Spring Boot import

## Configuration

Edit `ml/config.yaml` to customize:

```yaml
model:
  n_estimators: 100
  random_state: 42

evaluation:
  top_k_rates: [0.10, 0.20, 0.30]
```

## ML Job Worker

The worker polls `ml.ml_jobs` table and executes ML pipelines.

```bash
# Set DATABASE_URL once (or add to .env)
export DATABASE_URL=postgresql://postgres:postgres@localhost:5432/autolift_db

# Run worker
conda activate autolift-ml
python ml/scripts/run_ml_worker.py

# Or inline
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/autolift_db python ml/scripts/run_ml_worker.py
```
